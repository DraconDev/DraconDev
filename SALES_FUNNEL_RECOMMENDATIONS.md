# Sales Funnel Recommendations for DraconDev

Date: 2026-06-11

## Top 3 patterns to copy

1. **Direct product/course bio + destination link** — Wes Bos, Kent C. Dodds, Brad Traversy, and Brian Holt make the funnel obvious in the profile bio and route visitors to a course or product site.
2. **Founder/company funnel** — Rauchg and Amir Salihefendic turn the profile into a company portal by naming the company and linking to the product site.
3. **Teacher/platform funnel** — 1Marc, Brian Holt, and Kyle Simpson use the profile to route visitors to a course platform.

## Top 3 patterns to avoid

1. **Vague personal brand with no destination** — Josh Comeau and Addy Osmani are strong personal brands, but the profile itself does not sell anything clearly.
2. **Company link without product clarity** — a generic company link is weaker than a product-led destination.
3. **Too much code, not enough destination** — the best funnels are short, explicit, and destination-led.

## Concrete README changes to consider later

- Add a one-line hero that says what DraconDev sells or ships.
- Add a single obvious destination link to `dracon.uk` or a specific product page.
- Keep the Code section separate from product destinations.
- If you want a softer funnel, add a newsletter or YouTube CTA in the footer, not in the Code section.
- If you want a stronger funnel, promote one product or course at a time and make the destination explicit.

## Best-fit DraconDev pattern

The best fit is a hybrid of **Wes Bos + Kent C. Dodds + Brad Traversy**: short bio, explicit product/course destination, and a clear CTA. If DraconDev wants to sell tools, `dracon.uk` should be the destination; if DraconDev wants to sell learning content, a course or YouTube CTA should be the destination.
